## SECTION 3 — FALLBACK SYSTEM ARCHITECTURE (CRITICAL)

This is the most important section. Every tool MUST have 5–6 working fallbacks.

### 3.1 — What is a Fallback?
A fallback is an alternative method/library/service that automatically activates when the primary method fails. The user should NEVER see a raw error — they should either get a successful result (from a fallback) or a helpful message explaining what to try next.

### 3.2 — Fallback Priority Order for Every Tool
Each tool must implement this pattern:
1. **Primary Method** — The best, fastest, most reliable option (e.g., a high-quality library)
2. **Fallback 1** — A different library/approach that does the same thing
3. **Fallback 2** — A simpler/lighter version of the same operation
4. **Fallback 3** — A server-side alternative if client-side fails
5. **Fallback 4** — A third-party API if local processing fails
6. **Fallback 5** — A degraded-but-functional version (lower quality but working result)
7. **Final Fallback** — Clear error message with specific instructions for the user

### 3.3 — Fallback for Video Download Tools (URGENT — biggest problem area)
Current problem: Video downloads fail often. Fix using this fallback chain:
1. **Primary:** yt-dlp (latest version, kept updated) — direct stream download
2. **Fallback 1:** youtube-dl as alternative extractor
3. **Fallback 2:** cobalt.tools API (free, no auth needed)
4. **Fallback 3:** y2mate/similar API integration
5. **Fallback 4:** Direct MP4 stream extraction from manifest files
6. **Fallback 5:** Audio-only download (if video fails, offer MP3)
7. **Final:** Clear error with reason + suggestion to try again or use alternate format

For each video platform (YouTube, Instagram, TikTok, Twitter/X, Facebook, etc.), research and implement platform-specific extractors as fallbacks.

Auto-update yt-dlp on server startup or via cron job — outdated yt-dlp is the #1 reason for download failures.

### 3.4 — Fallback for PDF Tools
1. **Primary:** Ghostscript (for compression/conversion)
2. **Fallback 1:** pdf-lib (JavaScript library)
3. **Fallback 2:** PDFKit
4. **Fallback 3:** ilovepdf API (free tier)
5. **Fallback 4:** pdfcpu (Go library via subprocess)
6. **Fallback 5:** PyPDF2/pypdf (Python library)

### 3.5 — Fallback for Image Processing Tools
1. **Primary:** Sharp (Node.js — fastest)
2. **Fallback 1:** Jimp (pure JavaScript)
3. **Fallback 2:** ImageMagick (via subprocess)
4. **Fallback 3:** Canvas API (browser-side processing)
5. **Fallback 4:** Pillow via Python subprocess
6. **Fallback 5:** Browser's native canvas operations

### 3.6 — Fallback for OCR Tools
1. **Primary:** Tesseract.js (browser-based)
2. **Fallback 1:** Tesseract via Python subprocess
3. **Fallback 2:** EasyOCR (Python)
4. **Fallback 3:** Google Vision API (if configured)
5. **Fallback 4:** OCR.space free API
6. **Fallback 5:** Mathpix API (for math content)

### 3.7 — Fallback for Audio/Video Conversion
1. **Primary:** FFmpeg (server-side)
2. **Fallback 1:** ffmpeg.wasm (browser-side)
3. **Fallback 2:** Fluent-ffmpeg with different encoder settings
4. **Fallback 3:** Alternative codec/container
5. **Fallback 4:** Cloudconvert API free tier
6. **Fallback 5:** Offer download in alternative format

### 3.8 — Fallback for Background Removal
1. **Primary:** Remove.bg API (if API key configured)
2. **Fallback 1:** rembg (Python library)
3. **Fallback 2:** @imgly/background-removal (browser-based, no API key)
4. **Fallback 3:** Hugging Face model inference
5. **Fallback 4:** OpenCV-based segmentation
6. **Fallback 5:** Manual selection tool fallback

### 3.9 — Fallback for Text-to-Speech
1. **Primary:** Google TTS
2. **Fallback 1:** Microsoft Azure TTS
3. **Fallback 2:** Amazon Polly
4. **Fallback 3:** Web Speech API (browser native)
5. **Fallback 4:** pyttsx3 (Python, server-side)
6. **Fallback 5:** gTTS Python library

### 3.10 — General Fallback Rules
- Always wrap primary + all fallbacks in individual try/catch blocks
- Log which fallback was used (server-side logging, not visible to user)
- Never show technical error messages to users
- If ALL fallbacks fail, show: "This tool is temporarily unavailable. Please try again in a moment or try a different file format."
- Implement retry logic: auto-retry primary method once before moving to fallback
- Set timeout for each method (e.g., 30 seconds max per attempt)

## SECTION 5 — PERFORMANCE IMPROVEMENTS

### 5.1 — Frontend Performance
- Implement code splitting: Each tool's JavaScript should load only when that tool page is opened
- Lazy load images and non-critical scripts
- Use service workers for offline support and caching
- Compress all static assets (images, fonts, CSS, JS)
- Use WebP for all images on the site itself
- Implement CDN for static assets
- Remove all unused CSS (there will be a lot after 1300 tools)
- Minimize DOM operations — use virtual DOM or efficient rendering where possible

### 5.2 — Backend Performance
- Implement Redis caching for repeated operations (e.g., currency rates, repeated file conversions)
- Use worker threads for CPU-intensive operations (PDF processing, video conversion)
- Implement job queues (Bull/BullMQ) for heavy operations so server doesn't block
- Auto-scale file processing workers based on load
- Clean up temporary files aggressively (max 1 hour retention after last access)
- Implement request queuing to handle traffic spikes

### 5.3 — Database/Storage
- If no database is used yet, add one for: user sessions, tool usage analytics, error logging
- Use lightweight SQLite for simple storage needs, PostgreSQL for complex needs
- Never store user files beyond the processing session

### 5.4 — API Rate Limiting
- Implement per-IP rate limiting for all API endpoints
- Implement per-tool rate limiting (some tools like video downloader should have stricter limits)
- Show user-friendly "please wait" message when rate limited
- Implement exponential backoff for external API calls

## SECTION 6 — ERROR HANDLING STANDARDS

### 6.1 — User-Facing Errors (Never show technical details)
Every error message to users must:
- Be in simple, friendly language
- Explain what went wrong in one sentence
- Suggest what to try next
- Have a "Try Again" button
- Never show stack traces, error codes, or technical details

### 6.2 — Error Categories and Messages
- **File too large:** "Your file is too large. Please use a file under [X]MB, or compress it first using our [tool link]."
- **Wrong file format:** "This tool only supports [format list]. Please convert your file first using our [conversion tool link]."
- **Processing failed:** "We couldn't process this file. It may be corrupted. Please try a different file or contact support."
- **Network error:** "Connection lost. Please check your internet and try again."
- **Tool unavailable:** "This tool is temporarily busy. Please try again in a moment."
- **File protected:** "This file is password-protected. Please unlock it first using our [Unlock PDF tool]."
- **Video unavailable:** "This video is unavailable for download. It may be private, age-restricted, or region-blocked."

### 6.3 — Server-Side Error Logging
- Log every error with: timestamp, tool name, error type, user agent, file type/size (not file content)
- Alert on high error rates (>5% errors for any tool in any 10-minute window)
- Weekly error report to track most problematic tools

## SECTION 13 — TECHNOLOGY STACK RECOMMENDATIONS

### Current Stack (assumed): Next.js / React with Node.js backend
If different, adjust accordingly. These are the recommended libraries for each area:

### PDF Processing:
- **Primary:** pdf-lib, pdf-parse, Ghostscript (server-side)
- **Fallback:** PDFKit, jsPDF, pdfcpu, PyPDF2, iLovePDF API

### Image Processing:
- **Primary:** Sharp (Node.js — fastest)
- **Fallback:** Jimp, Canvas API, ImageMagick, Pillow (Python), browser Canvas

### Video/Audio:
- **Primary:** FFmpeg server-side, yt-dlp
- **Fallback:** ffmpeg.wasm (browser), youtube-dl, cobalt.tools API, cloudconvert API

### OCR:
- **Primary:** Tesseract.js + tessdata
- **Fallback:** EasyOCR (Python), Google Vision API, OCR.space API

### Background Removal:
- **Primary:** @imgly/background-removal (browser, no API key needed)
- **Fallback:** rembg (Python), remove.bg API, Hugging Face model

### Text-to-Speech:
- **Primary:** Web Speech API (browser) + gTTS
- **Fallback:** Google TTS API, Azure TTS, Amazon Polly, espeak

### Machine Learning (for smart features):
- **Primary:** TensorFlow.js (browser-side, no server needed)
- **Fallback:** ONNX Runtime Web, Hugging Face Inference API (free tier)

### Real-time Exchange Rates:
- **Primary:** Open Exchange Rates (free tier)
- **Fallback:** Fixer.io, exchangerate-api.com, Currency-API (GitHub free), ECB data

### Caching:
- **Primary:** Redis
- **Fallback:** node-cache, memory-cache, LRU-cache

### Job Queue:
- **Primary:** BullMQ with Redis
- **Fallback:** bee-queue, kue, in-memory queue
