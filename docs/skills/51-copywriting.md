# Copywriting Skill — Conversion-Focused Marketing Copy

## What It Does
Writes high-converting marketing copy for websites, landing pages, pricing pages, feature pages, about pages, CTAs, taglines, and value propositions. Designed to persuade, convert, and communicate product value clearly.

## Context-First Approach

Before writing, check for existing context at `.agents/product-marketing-context.md`. If it exists, read it first. Then gather only missing context:

### Information to Gather
1. **Page type**: Homepage, landing page, pricing, feature, about
2. **Product**: What it does, key differentiator
3. **Target audience**: Who uses it, what they struggle with
4. **Tone**: Professional, casual, bold, playful, serious
5. **Goal**: Sign up, purchase, download, contact
6. **Competitors**: What makes this different

## Copy Frameworks Used

### AIDA (Awareness → Interest → Desire → Action)
Best for landing pages with a clear funnel.

### PAS (Problem → Agitate → Solve)
Best for pain-point-driven products.

### FAB (Feature → Advantage → Benefit)
Best for technical products with specific capabilities.

### Jobs-to-be-Done
Focus on what the user is trying to accomplish, not the product's features.

## Structure for Different Page Types

### Homepage Hero Section
```
[Eyebrow Tag] — Sets context (e.g., "100% Free · No Signup Required")
[H1 Headline] — Clear, benefit-driven, under 10 words
[Subheadline] — 1-2 sentences expanding the promise
[CTA Button] — Action verb + outcome (e.g., "Start for Free →")
[Social Proof] — Users count, rating, or logos
```

### Feature Page
```
[Feature Name + Icon]
[Benefit Headline] — What it does for the user
[2-3 bullet points] — Specific capabilities
[Screenshot/Demo]
[Mini-CTA]
```

### Pricing Page
```
[Plan Name] — Intuitive names (Starter, Pro, Enterprise)
[Price + Period]
[For who] — "Perfect for freelancers / teams / enterprise"
[Feature list] — 5-7 items, benefits not just features
[CTA]
```

## ISHU TOOLS Specific Copy Guidelines

ISHU TOOLS serves:
- **Indian students** → Use relatable language, mention "free", "no signup", "works offline"
- **Everyday users** → Keep it simple, avoid jargon
- **SEO-first** → Every page headline should include the tool name + key benefit

### Example Headlines for Tools
- "Compress PDF — Reduce File Size Instantly, 100% Free"
- "Instagram Downloader — Save Instagram Videos Without Watermark"
- "BMI Calculator India — Find Your Healthy Weight Range"

## Quality Checklist
- [ ] Clear what the product does in first 5 seconds
- [ ] Benefit-driven (not feature-driven)
- [ ] One clear CTA per section
- [ ] No jargon or corporate-speak
- [ ] Addresses main objection/fear
- [ ] Includes social proof if available
- [ ] Mobile-readable (short paragraphs, scannable)
