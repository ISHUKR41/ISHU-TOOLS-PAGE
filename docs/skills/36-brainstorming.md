# Brainstorming Skill (User-Provided)

## What It Does
Explores user intent, requirements, and design BEFORE implementation. MUST be used before any creative work — creating features, building components, adding functionality, or modifying behavior. Prevents building the wrong thing.

## CRITICAL: Use Before ANY Creative Work
This skill should activate before:
- Creating new features or tools
- Building UI components
- Adding functionality
- Modifying app behavior significantly

## Why This Matters
Jumping directly to implementation without understanding intent often results in:
- Building the wrong feature
- Missing important requirements
- Needing to redo work
- User frustration

## Brainstorming Process
1. **Explore** — Ask clarifying questions about the user's goal
2. **Requirements** — Extract functional and non-functional requirements
3. **Design** — Consider multiple approaches before committing
4. **Decision** — Choose the best approach based on constraints
5. **Implement** — Now build with full context

## Questions to Explore
- What problem is this solving? For whom?
- What does success look like?
- Are there examples or references to follow?
- What constraints exist (tech, time, budget)?
- What should NOT be changed?
- How will users discover/use this?

## For ISHU TOOLS Feature Requests
When user asks for new tools:
1. What specifically should the tool do?
2. What input does it accept?
3. What output should it produce?
4. Are there quality/accuracy requirements?
5. Any specific format preferences?
6. Reference websites showing similar tools?
